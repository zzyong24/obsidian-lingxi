# Obsidian AI Chat Plugin — 安装部署指南

> 本文档面向开发者和高级用户，详细说明如何从源码构建、安装和配置 AI Chat 插件。

---

## 1. 环境要求

| 依赖 | 最低版本 | 推荐版本 | 说明 |
|------|---------|---------|------|
| Node.js | 16.x | 18.x 或 20.x | 构建工具运行环境 |
| npm | 8.x | 9.x+ | 包管理器（Node.js 自带） |
| Obsidian | 0.15.0 | 1.5.0+ | 目标运行平台 |
| TypeScript | 5.x | 5.7+ | 开发依赖，自动安装 |

### 检查环境

```bash
node -v   # 应显示 v16.x 或更高
npm -v    # 应显示 8.x 或更高
```

---

## 2. 获取源码

```bash
# 克隆仓库
git clone <仓库地址>
cd obsidian-lingxi

# 或者直接在已有工程中进入项目目录
cd /path/to/obsidian-lingxi
```

---

## 3. 安装依赖

```bash
npm install
```

这会安装以下关键依赖：

| 依赖 | 用途 |
|------|------|
| `react` / `react-dom` | UI 渲染框架 |
| `react-markdown` | Markdown 内容渲染 |
| `uuid` | 生成唯一 ID |
| `obsidian` | Obsidian API 类型定义（开发依赖） |
| `esbuild` | 打包构建工具（开发依赖） |
| `typescript` | TypeScript 编译器（开发依赖） |

---

## 4. 构建

### 4.1 生产构建

```bash
npm run build
```

该命令依次执行：
1. `tsc -noEmit -skipLibCheck` — TypeScript 类型检查（不输出文件）
2. `node esbuild.config.mjs production` — esbuild 打包，输出压缩后的 `main.js`

构建成功后，项目根目录下会生成 `main.js` 文件。

### 4.2 开发模式

```bash
npm run dev
```

启动 esbuild 的 watch 模式，修改源码后自动重新构建（带 sourcemap）。配合 Obsidian 的「Reload app without saving」（`Ctrl/Cmd + R`）即可热调试。

---

## 5. 安装到 Obsidian

### 方式一：手动安装（推荐开发调试）

1. 打开你的 Obsidian Vault 所在文件夹

2. 进入（或创建）插件目录：
   ```bash
   mkdir -p <你的Vault路径>/.obsidian/plugins/ai-chat
   ```

3. 将以下 3 个文件复制到该目录：
   ```bash
   cp main.js     <你的Vault路径>/.obsidian/plugins/ai-chat/
   cp styles.css   <你的Vault路径>/.obsidian/plugins/ai-chat/
   cp manifest.json <你的Vault路径>/.obsidian/plugins/ai-chat/
   ```

4. 重启 Obsidian（或按 `Ctrl/Cmd + R` 重载）

5. 进入 **设置 → 第三方插件**，找到「AI Chat」，点击启用

### 方式二：符号链接（开发最佳实践）

在开发过程中，可以创建符号链接避免反复复制文件：

```bash
# macOS / Linux
ln -s /Users/zyongzhu/workbase/github/obsidian-knowledge/obsidian-lingxi /Users/zyongzhu/workbase/github/obsidian-knowledge/obsidian-knowledge/.obsidian/plugins/ai-chat

# Windows (需要管理员权限)
mklink /D "<你的Vault路径>\.obsidian\plugins\ai-chat" "<项目路径>"
```

这样源码目录中的 `main.js`、`styles.css`、`manifest.json` 会直接被 Obsidian 读取。配合 `npm run dev` 的 watch 模式，修改代码后只需在 Obsidian 中按 `Ctrl/Cmd + R` 即可看到效果。

### 方式三：BRAT 插件安装（适合测试用户）

如果项目已发布到 GitHub：

1. 在 Obsidian 中安装 [BRAT](https://github.com/TfTHacker/obsidian42-brat) 插件
2. 打开 BRAT 设置 → Add Beta Plugin
3. 输入仓库地址，点击 Add
4. BRAT 会自动下载最新 Release 并安装

## 6. 首次配置

### 6.1 配置模型 API Key

1. 打开 Obsidian **设置 → AI Chat**
2. 展开你想使用的模型提供商（如 DeepSeek）
3. 填写 **API Key**
4. 可选：修改 Base URL 和默认模型名称
5. 点击「测试连接」确认配置正确

> 💡 **提示**：至少需要配置一个提供商的 API Key 才能开始使用。

**各提供商 API Key 申请地址**：

| 提供商 | 申请地址 | 免费额度 |
|--------|---------|---------|
| DeepSeek | [platform.deepseek.com](https://platform.deepseek.com/) | 新用户赠送额度 |
| 通义千问 | [dashscope.console.aliyun.com](https://dashscope.console.aliyun.com/) | 有免费调用额度 |
| 豆包 | [console.volcengine.com/ark](https://console.volcengine.com/ark) | 新用户赠送额度 |
| Kimi | [platform.moonshot.cn](https://platform.moonshot.cn/) | 有免费调用额度 |
| 智谱 | [open.bigmodel.cn](https://open.bigmodel.cn/) | 新用户赠送额度 |

### 6.2 选择默认模型

在设置面板「默认模型」区域：
- **默认文本模型**：选择一个已配置 API Key 的模型（如 `DeepSeek / deepseek-chat`）
- **默认视觉模型**：如需图片分析能力，选择支持视觉的模型

### 6.3 创建 Skill 和 Rules 文件夹

在你的 Vault 根目录下创建以下文件夹：

```
My Vault/
├── _skills/         # Skill 模板文件夹
├── _rules/          # 全局规则文件夹
└── _ai_output/      # AI 输出归档（会自动创建）
```

> 文件夹名称可以在设置中自定义。

### 6.4 创建你的第一个 Skill

在 `_skills/` 文件夹中新建一个 `.md` 文件，例如 `选题生成.md`：

```markdown
---
name: 选题生成
description: 根据主题和目标受众，生成短视频选题创意
trigger_keywords: ["选题", "创意", "idea"]
category: 内容创作
output_folder: _ai_output/选题
output_template: note
model_preference: text
---

## System Prompt

你是一个资深的短视频内容策划师，擅长为不同平台（抖音、小红书、视频号）生成爆款选题。

用户会告诉你一个主题方向和目标受众，你需要：
1. 生成 5 个具体的选题创意
2. 每个选题包含：标题、核心卖点、目标情绪、建议的内容结构
3. 标注每个选题适合的平台

## 输出格式

请按以下结构输出每个选题：

### 选题 N：{标题}
- **核心卖点**：...
- **目标情绪**：...
- **内容结构**：...
- **适合平台**：...
```

### 6.5 创建全局规则（可选）

在 `_rules/` 文件夹中新建 `.md` 文件，例如 `输出规范.md`：

```markdown
## 通用输出规范

- 使用中文输出
- 语气活泼、接地气，适合短视频创作者阅读
- 使用 Markdown 格式，结构清晰
- 每次输出结尾附上 3 个相关的 #标签
```

此规则会在**每次对话**中自动注入到 System Prompt 最前面。

## 7. 使用方式

### 打开聊天面板

以下任一方式：
- 点击左侧 Ribbon 栏的 💬 图标
- 命令面板（`Ctrl/Cmd + P`）→ 输入「打开 AI Chat」
- 快捷键（如已设置）

### 普通对话

直接在输入框输入消息，按 Enter 发送（或 Ctrl+Enter，取决于设置）。

### 使用 Skill

1. 输入 `/` 弹出 Skill 选择面板
2. 通过搜索或键盘上下键选择 Skill
3. 按 Enter 确认，输入框上方出现 Skill 标签
4. 输入内容并发送，AI 将按 Skill 定义的 System Prompt 回复
5. 回复完成后自动归档到指定文件夹

### 切换模型

点击聊天界面顶部的模型名称，在下拉菜单中选择其他已配置的模型。

### 保存对话内容

- **自动归档**：使用 Skill 时，AI 回复自动保存为笔记
- **手动保存**：将鼠标悬停在 AI 回复上，点击「💾 保存为笔记」按钮
- **复制内容**：点击「📋 复制」按钮复制到剪贴板

## 8. 开发调试

### 开启开发者工具

打开 Obsidian 开发者控制台，可查看插件日志（`[AI Chat]` 前缀）：

- **macOS**：按 `Cmd + Option + I`，或通过命令面板搜索「Toggle Developer Tools」
- **Windows / Linux**：按 `Ctrl + Shift + I`
- **通用**：命令面板（`Ctrl/Cmd + P`）→ 输入「Toggle Developer Tools」

### 开发工作流

```bash
# 1. 创建符号链接（一次性）
ln -s /path/to/obsidian-lingxi <Vault>/.obsidian/plugins/ai-chat

# 2. 启动 watch 模式
npm run dev

# 3. 在 Obsidian 中启用插件

# 4. 修改代码后，在 Obsidian 中按 Ctrl/Cmd + R 重载
```

### 项目脚本

| 命令 | 说明 |
|------|------|
| `npm run dev` | 开发模式（watch + sourcemap） |
| `npm run build` | 生产构建（类型检查 + 压缩打包） |
| `npm run version` | 版本号更新（自动修改 manifest.json） |

项目配置了 `@/*` → `src/*` 的路径别名，在代码中可使用：

```typescript

```

## 9. 发布部署

### 打包发布文件

```bash
# 生产构建
npm run build

# 需要发布的 3 个文件
# - main.js         构建产物
# - styles.css       样式文件
# - manifest.json    插件元信息
```

### GitHub Release

1. 更新 `manifest.json` 和 `package.json` 中的版本号
2. 更新 `versions.json` 中的版本映射
3. 运行 `npm run build`
4. 创建 GitHub Release，上传 `main.js`、`styles.css`、`manifest.json`

### 提交到 Obsidian 社区插件市场

1. Fork [obsidian-releases](https://github.com/obsidianmd/obsidian-releases) 仓库
2. 在 `community-plugins.json` 中添加插件信息
3. 提交 Pull Request
4. 等待 Obsidian 团队审核

## 10. 常见问题

### Q: 提示「请先配置模型 API Key」？

进入 **设置 → AI Chat**，至少为一个模型提供商配置有效的 API Key，并在「默认模型」区域选择该提供商。

### Q: Skill 文件没有被加载？

检查以下几点：
1. Skill 文件是否在设置中指定的 Skill 文件夹下（默认 `_skills/`）
2. 文件是否为 `.md` 格式
3. 文件是否包含正确的 Frontmatter（`---` 包裹的 YAML 头部）
4. 文件是否包含 `## System Prompt` 章节
5. 打开开发者控制台，查看 `[AI Chat]` 相关日志

### Q: 流式输出不工作？

部分网络环境可能拦截 SSE 连接。尝试：
1. 在设置中关闭「流式输出」，使用非流式模式
2. 检查网络代理设置
3. 确认 API Key 有效且有剩余额度

### Q: 归档的文件在哪里？

默认归档到 Vault 根目录下的 `_ai_output/` 文件夹。如果 Skill 设置了 `output_folder`，则归档到对应文件夹。所有路径均可在设置中修改。

### Q: 如何使用自定义模型（如本地部署的 LLM）？

在设置面板中修改任一提供商的 Base URL 为你的本地服务地址（如 `http://localhost:11434/v1`），只要该服务兼容 OpenAI API 格式即可。
